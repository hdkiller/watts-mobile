# 033 — Today generation poll: setInterval leaks on unmount and stacks on repeat taps

**Area:** Today · **Priority:** medium · **Effort:** S · **Status:** fixed (2026-07-20)

## Problem
`onGenerate` in [today.tsx](<../../app/(app)/(tabs)/today.tsx>) starts a `setInterval(…, 2500)` that:

- is stored only in a local variable — leaving the screen never clears it; it keeps polling `/api/recommendations/status` (and calling `setGenState` on an unmounted screen) for up to 75s;
- is created per call — tapping "Analyze Readiness" again after an error (or fast double-tap) runs multiple concurrent pollers;
- runs an async callback on a fixed interval, so slow responses overlap;
- aborts to a hard error on the **first** failed status check, so one transient network blip while generation is actually still running shows "Failed to check generation status."

## Proposal
Keep the interval id in a ref, clear it in a `useEffect` cleanup and at the start of every `onGenerate`; guard re-entry while `genState === 'generating'`. Tolerate N consecutive status failures before surfacing an error. (Longer term this belongs in a `useQuery` with `refetchInterval`, same as `useActivitySummaryQuery`'s analyzing poll.)
