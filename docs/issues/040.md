# 040 — Bottom content trapped under the floating native tab bar

**Area:** tabs/layout · **Priority:** medium · **Effort:** S · **Status:** done

## Problem
The app uses iOS 26 `NativeTabs` ([(tabs)/_layout.tsx](<../../app/(app)/(tabs)/_layout.tsx>)) — a floating, translucent bar overlaying content — but tab screens pad their scroll content with fixed values that don't clear it: Log `paddingBottom: 48`, Today `pb-10`, More `pb-12`. Verified live:

- **More**: the "Privacy policy" row sits permanently behind the tab bar glass.
- **Log › Nutrition**: the hydration total line at the bottom can't scroll clear of the bar.
- **Today**: the "How you're doing" strip is half-covered at rest.

Fixed padding also ignores the home-indicator inset, so the shortfall varies by device.

## Proposal
One shared constant/hook for tab screens: bottom padding = tab-bar height + bottom safe-area inset (use `useSafeAreaInsets().bottom` + ~64, or switch the ScrollViews to `contentInsetAdjustmentBehavior="automatic"` and let UIKit do it — NativeTabs propagates the inset). Apply to Today, Log, More scroll containers; keep the Log keyboard `overlap` additive.
