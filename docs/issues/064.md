# 064 — Persist Health Sync activity metrics on wellness (`steps`, distance, floors, exercise minutes)

**Status:** done in coach-wattz schema/API (migration pending deployment, 2026-07-21)  
**Area:** health sync / API  
**Related:** `openspec/changes/health-platform-ingest-v2`

## Problem

Mobile Health Sync now uploads daily activity-volume fields as first-class JSON keys:

- `steps`
- `distanceMeters`
- `exerciseMinutes`
- `floors`

`POST /api/wellness` validates with Zod (`wellnessUploadSchema` in `server/api/wellness/index.post.ts`). Unknown keys are **stripped**, not rejected — so the request succeeds but those metrics never land on the `Wellness` row (Prisma has no columns for them either). Steps shipped in ingest v2 are therefore lost on the server today.

## Desired

1. Extend `wellnessUploadSchema` (and patch schema) to accept the four fields.
2. Add matching Prisma `Wellness` columns (or a documented home in `customMetrics` / `rawJson.activity` that the product UI reads).
3. Surface them in Today wellness glances / analysis where useful.

## Resolution

coach-wattz now accepts and persists all four fields on `Wellness`; mobile continues mirroring them under `rawJson.activity` for provenance and backward compatibility with older self-hosted instances.
