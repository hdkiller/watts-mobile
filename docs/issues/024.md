# 024 — Offline-first Today and planned detail

**Area:** expansion · **Priority:** medium · **Effort:** M · **Status:** done

## Problem
At a trailhead with no signal, Today and the planned workout structure are unreadable; baseline lists "stronger offline Today" for later.

## Proposal
Persist the TanStack Query cache (async-storage persister) for Today + planned detail queries; render cached data with a "last updated 7:02" stamp and an offline banner instead of the error card when the network is down.
