# 034 — Check-in form state clobbered by wellness refetch

**Area:** Log · **Priority:** medium · **Effort:** S · **Status:** open

## Problem
[log.tsx](<../../app/(app)/(tabs)/log.tsx>) resets the whole form whenever the today-wellness query delivers data:

```ts
useEffect(() => {
  if (todayWellness) {
    setValues(formFromWellness(todayWellness));
    …
  }
}, [todayWellness]);
```

Any refetch that resolves while the athlete is mid-edit — save-triggered invalidation racing a kept-typing user, pull-to-refresh on Today invalidating `['wellness']`, remount refetch after `staleTime` — replaces sliders and half-typed sleep/notes with the server snapshot. Sliders silently snap back; typed text vanishes.

## Proposal
Hydrate the form only when it's pristine: track a `dirty` flag (set in `touchForm`, cleared on successful save) and skip the effect when dirty. Or hydrate once per screen focus and after `onSave` explicitly, instead of reacting to every query delivery.
