# 036 — Week glance bars mix seconds with TSS, flattening planned days

**Area:** Today · **Priority:** low · **Effort:** XS · **Status:** open

## Problem
[weekGlance.ts:99–111](../../src/features/today/weekGlance.ts) builds bar heights from `doneByDay` (duration **seconds**, e.g. 3600 for a 1h ride) plus `plannedByDay` (**TSS**, ~50–100) × 0.35. Once any completed day exists, `maxLoad` is in the thousands and planned-only days render bars of ~1% height — visually zero. The `hasPlanned` dot still shows, but the "relative load" chart the strip promises is meaningless across done vs planned days.

## Proposal
Normalize both series into one unit before mixing — e.g. use TSS for done days when present (falling back to `durationSec / 36` ≈ TSS-ish), or scale each map to its own max before combining.
