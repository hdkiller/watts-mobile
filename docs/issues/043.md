# 043 — Raw workout type strings in UI ("WeightTraining", "VirtualRun")

**Area:** cross-cutting · **Priority:** low · **Effort:** XS · **Status:** done

## Problem
List rows and meta lines print the server's type enum verbatim: "Tue, Jul 21 · **WeightTraining** · 1h" (Coming up, Upcoming, Recently), and notification titles like "New Run, VirtualRun, TrailRun FTP Detected". CamelCase enum soup in athlete-facing copy reads as unfinished.

## Proposal
Add a `humanizeWorkoutType(type)` helper (split camelCase, capitalize first word: "Weight training", "Virtual run"; keep known acronyms) and use it wherever `type` is displayed — Today teasers, upcoming/recent lists, planned/activity detail meta, widget meta line. Server-authored notification copy stays as-is (fix belongs in coach-wattz).
