# 039 — Check-in card promises questions; screen says "No questions today"

**Area:** Today/check-in · **Priority:** high · **Effort:** S · **Status:** open

## Problem
Verified live: Today shows the highlighted "Coach Check-In" card — "**Coach has questions prepared** to adjust today's recommendation" — but tapping it lands on an empty screen: "No questions today · There are no checklist items required for your session today" with a full-width **primary** "Go Back" button. Three frictions:

1. **Broken promise.** The shared helper [isDailyCheckinCompleted.ts](../../src/features/log/isDailyCheckinCompleted.ts) (used by Today's card condition) returns `false` when `questions.length === 0`, so a check-in with zero questions counts as "not completed" and the card shows indefinitely — advertising questions that don't exist. The fix point is that helper (or a distinct "nothing needed" state alongside it).
2. **Dead-end CTA.** The strongest visual in the app (brand-green primary button) is spent on "Go Back". Primary styling should be reserved for forward actions.
3. The empty screen is a vast black void with two centered lines — no skeleton-matching layout, no alternative action (e.g. "Generate questions" / open wellness check-in instead).

## Proposal
Treat a zero-question check-in as complete (or as "nothing needed") in the Today card condition so the card only renders when there are unanswered questions. On the empty screen, demote "Go Back" to secondary and offer a useful forward path (generate check-in if the API allows `force`, or link to the wellness check-in). Copy: "Nothing to check in today — Coach will ask when it matters."
