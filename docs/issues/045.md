# 045 — Coach open uses a full-screen spinner, against the app's own skeleton rule

**Area:** Coach · **Priority:** low · **Effort:** S · **Status:** open

## Problem
[CoachChat.tsx](../../src/features/coach/CoachChat.tsx) shows a centered `ActivityIndicator` + "Opening Coach…" for the initial load. [DESIGN.md](../DESIGN.md) is explicit: "Skeletons, not spinners — full-screen loads show layout-matching skeletons"; every other screen complies. Chat is also one of the slower opens (rooms fetch → messages fetch sequentially), so the spinner is the state users actually see.

## Proposal
Add a chat skeleton (header bar + 3–4 alternating bubble placeholders + composer) using the shared `Skeleton` component, replacing the spinner branch. Bonus: the sequential `fetchChatRooms` → `fetchChatMessages` bootstrap could kick off the messages fetch for the likely room in parallel to shorten the wait it masks.
