# 055 — iPad is enabled but ships stretched phone layouts

**Area:** store/layout · **Priority:** medium · **Effort:** S–M · **Status:** open

## Problem
[app.json](../../app.json) sets `ios.supportsTablet: true`, but every screen is a phone layout: full-width `px-6` columns, `w-1/2` metric grids, tab bar + hero cards designed for ~390pt. On a 13" iPad that renders as comically wide cards and 40-character-per-line walls. Since tablet support is declared, App Store review runs the app on iPad and screenshots are required for the listing — the current experience undermines the otherwise tight design.

## Proposal
Cheapest: set `supportsTablet: false` for v1 (phone-only listing; iPad users get scaled phone mode) and note it in the store checklist. If tablet stays on: add a shared `max-w-[560px] self-center w-full` content container to scroll screens and let Coach chat/detail pages breathe — a one-component change that covers most screens. Decide before the next store submission.
