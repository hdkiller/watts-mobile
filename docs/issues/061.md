# 061 — Chat list: empty "New Chat" rooms pile up; previews say "No messages yet" on rooms with history

**Area:** Coach · **Priority:** medium · **Effort:** S · **Status:** done

## Problem
Live in the room switcher, two related issues made the list unusable for finding a past conversation:

1. **Room clutter.** Eight-plus "New Chat · No messages yet" entries, several per day. The 15-minute session policy (`decideSessionOpen`) created a fresh room on every Coach-tab open when the last room was idle — including when the athlete just looked and never sent. Empty rooms persist server-side and accumulate forever.
2. **Broken previews.** Rooms that clearly contain conversations ("Mai pihenőnap", "Mai edzésterv keresése") also showed "No messages yet" — `previewForRoom` reads `room.lastMessage?.content`, and coach-wattz `rooms.get.ts` fills `content` from the message's `content` column, which is **empty for parts-based AI-SDK messages**.

## Resolution (2026-07-20)
Mobile side shipped: [sessionPolicy.ts](../../src/features/coach/sessionPolicy.ts) `decideSessionOpen` now reuses the newest **empty** room (no `lastMessage`) instead of creating another when the session is stale (tests updated), and [RoomListSheet.tsx](../../src/features/coach/RoomListSheet.tsx) `previewForRoom` shows "Conversation" when `lastMessage` exists with empty content instead of the false "No messages yet". Verified on device.

**Remaining server-side (coach-wattz):** `server/api/chat/rooms.get.ts` previews read the `content` column — needs a server fix to surface real preview text for parts-based messages; the existing pile of empty rooms could also be cleaned up server-side.
