# 010 — Hero stat blocks on detail screens

**Area:** detail · **Priority:** medium · **Effort:** S · **Status:** done

## Problem
On planned and activity detail, key numbers (duration, TSS, load) are buried in one `·`-joined metadata line.

## Proposal
Add 2–3 larger stat tiles (label + value) under the title on `app/(app)/planned/[id].tsx` and `app/(app)/activity/[id].tsx`; keep the joined line for secondary meta (date, type). Reuse the tile style from Today's recovery metrics.

## Done
- Shared `HeroStatTiles` (`src/components/HeroStatTiles.tsx`) — recovery-tile chrome, larger value type.
- Planned: Duration / TSS / IF under title; meta line is date · type.
- Activity: Duration / TSS (or Load) / IF under title; meta line is date · type.
