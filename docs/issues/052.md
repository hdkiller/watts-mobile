# 052 — Units & locale: raw always-expanded IANA timezone list

**Area:** settings · **Priority:** low · **Effort:** S · **Status:** done

## Problem
Live on [settings/units.tsx](<../../app/(app)/settings/units.tsx>): below the three tidy unit segments, the timezone control is a search field plus an **always-expanded raw IANA list** ("UTC", "America/Los_Angeles", "America/Denver", …) with underscores intact, pushing the screen to several viewports. The current selection ("Europe/Budapest (device)") and the "Use device timezone" action are the only things most athletes need.

## Proposal
Collapse the list behind the search field: show only "Selected: Europe/Budapest (device)" + "Use device timezone" + "Change…"; render the list (filtered as you type) only after the athlete taps Change or focuses search. Humanize display names (replace `_` with space, optionally show city + GMT offset). Keep the stored value as the IANA id.
