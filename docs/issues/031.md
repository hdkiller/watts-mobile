# 031 — Wellness check-in weight: display unit label ≠ stored/sent unit

**Area:** Log · **Priority:** high · **Effort:** S · **Status:** done

## Problem
The wellness check-in weight field is internally inconsistent for `Pounds` athletes ([log.tsx](<../../app/(app)/(tabs)/log.tsx>), [mapLogForm.ts](../../src/features/log/mapLogForm.ts), [applyHealthPrefill.ts](../../src/features/log/applyHealthPrefill.ts)):

- The field label uses the profile unit (`Weight (lbs)`).
- **Health prefill** converts kg → lb before filling (`applyHealthPrefill` with `weightUnit: 'lb'`) — so the field holds a lbs number.
- **Loading today's saved wellness** (`formFromWellness`) puts `day.weight` into the field **raw**, no conversion — presumably kg from the server.
- **Saving** (`toWellnessPayload`) posts the field value **raw** to `POST /api/wellness`, no unit conversion or unit field.

So the same field is kg on load, lbs after prefill, and whatever-it-holds on save. A `Pounds` athlete who types their weight as 165 either stores 165 kg server-side, or (if the server converts like the profile PATCH does) the load path shows kg under a lbs label. At least one of the three paths is wrong regardless of the API contract.

## Proposal
Check coach-wattz's wellness API contract (does `POST /api/wellness` expect kg or profile display units?). Then make all three paths agree: convert on load (`kgToDisplayWeight`) and convert back on save, mirroring `mapProfile`'s PATCH handling. Add mapLogForm tests for the Pounds round-trip.
