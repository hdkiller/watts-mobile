# 049 — Every drill-down hides the tab bar; journeys pile onto one global stack

**Area:** navigation · **Priority:** medium · **Effort:** M · **Status:** done

## Problem
All non-tab screens — activity list/detail, planned detail, upcoming, athlete, notifications, check-in, recovery event, every settings page — are registered as **siblings of `(tabs)` in the root Stack** ([(app)/_layout.tsx](<../../app/(app)/_layout.tsx>)). Two consequences, both felt repeatedly in live use:

1. **The tab bar disappears on every push.** From an activity detail or Settings page there is no way to jump to Today or Coach — only back-back-back. iOS convention keeps the tab bar visible for in-tab drill-downs (each tab owning its own stack); full-screen takeover is normally reserved for modal flows.
2. **One shared stack accumulates cross-tab history.** Upcoming → back → More → Settings → Notifications → back can land on a screen from a *previous* journey (I hit Settings → back → Athlete). Back becomes unpredictable the longer the session runs.

## Proposal
Move per-tab drill-downs into per-tab stacks (Expo Router: `(tabs)/today/_layout.tsx` etc., or `Stack` inside each tab group) so activity/planned/upcoming live under Today's stack, settings/athlete/notifications under More's, and the tab bar stays reachable. Keep true modals (check-in, recovery event) as `presentation: 'modal'` on the root. This is a routing refactor — deep-link hrefs in [pathMap.ts](../../src/linking/pathMap.ts)/`resolveDeepLink` must be updated in the same change.

## Resolution (2026-07-20)
Per-tab stacks shipped:

- **Today stack** (`(tabs)/today/`): index, planned `[id]`, activity list/detail, upcoming
- **More stack** (`(tabs)/more/`): index, athlete, notifications, settings/*, sports `[id]`
- **Log stack** (`(tabs)/log/`): index only
- **Root modals** remain: `daily-checkin`, `recovery-event`
- Deep-link / push targets updated via `APP_HREFS` + `migrateLegacyAppHref` (legacy `/(app)/activity|planned|…` still resolve)
- Docs: [docs/deep-links.md](../deep-links.md); OpenSpec `today-home` / `account-more` hrefs
