# 038 — Chat shows internal seed context as the athlete's own message

**Area:** Coach · **Priority:** high · **Effort:** S · **Status:** open

## Problem
The first message in a chat is sent with the seed prefix baked into the user text ([useCoachChat.ts:697](../../src/features/coach/useCoachChat.ts), [seedContext.ts](../../src/features/coach/seedContext.ts)), and `visibleCoachMessages` renders it verbatim. The athlete taps a starter like "Session intent" and sees a giant green bubble of their "own" words:

> Context for this chat (facts only — do not invent a new plan):
> Today's recommendation: …
> Athlete question: What's the intent of today's planned session…

Verified live on device. This leaks prompt plumbing into the UI, makes the first exchange feel broken, and the bubble dwarfs the actual question. It also persists — every reload of the room re-renders the seed wall at the top of history.

## Proposal
Keep sending the seed (server has no separate context channel today) but strip it for display: in `Bubble`/`messageText`, when a user message matches the `withSeedPrefix` shape, render only the text after `Athlete question:`. Do it in one place (a `displayText(message)` helper next to `messageText`) so history loads and live sends both benefit. Longer term, ask coach-wattz for a metadata/context field on `POST /api/chat/messages` so the seed never rides in the visible text.
