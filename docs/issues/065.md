# 065 — More → Recent activity / Upcoming: Back lands on Today

**Area:** navigation · **Priority:** high · **Effort:** S–M · **Status:** open

## Problem
More → Training & Schedule pushes into the **Today** tab stack:

- Recent activity → `/(app)/(tabs)/today/activity`
- Upcoming planned → `/(app)/(tabs)/today/upcoming`

Back pops to Today’s root, not More. Same class as the fixed Today → Athlete bug (athlete moved to root `/(app)/athlete`).

Product baseline still lists these lists under More; the live routes were nested under Today in the **049** per-tab restructure without preserving the More back target.

## Proposal
Prefer the athlete pattern: hoist multi-entry list (and detail) destinations to the **root app stack**, or otherwise ensure Back from a More-originated open returns to More. Keep deep-link path map stable (`/activities`, `/upcoming`) via `APP_HREFS` + `migrateLegacyAppHref`.
