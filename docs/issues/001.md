# 001 — Friendly error copy instead of raw API messages

**Area:** cross-cutting · **Priority:** high · **Effort:** S · **Status:** done

## Problem
Several screens render `error.message` straight from the API/fetch layer (Today error card, activity/upcoming/notifications lists, detail screens). Network failures surface as developer strings.

## Proposal
Add a small `friendlyError(err, fallback)` helper (e.g. `src/api/errors.ts`) that maps common cases — offline, timeout, 401/403, 5xx — to human copy ("Can't reach your Coach Watts instance — check your connection"). Keep the raw message for Sentry breadcrumbs only. Sweep all `error instanceof Error ? error.message : …` call sites.

**OpenSpec change:** `login-and-error-polish`

## Resolution
Shipped in `feat/login-and-error-polish`: `src/api/errors.ts` + vitest; call sites (Today, Log, activity, planned, upcoming, notifications, coach chat, nutrition, athlete, recovery) use `friendlyError`. Airplane-mode / device visual check remains as OpenSpec task 3.2.
