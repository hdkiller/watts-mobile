# 006 — Faster check-in: tap controls, save feedback, weight unit

**Area:** Log · **Priority:** high · **Effort:** M · **Status:** in progress

## Problem
Readiness (1–10) and sleep quality (1–5) are keyboard text inputs — the biggest friction in the "<30s morning path". The disabled Save button already dims (done), but "Saved for today." is easy to miss, the label never becomes "Update check-in", and the weight placeholder hard-codes "kg".

## Proposal
In `app/(app)/(tabs)/log.tsx`:
- Replace readiness and sleep-quality fields with tappable number-chip rows (no keyboard); stepper for sleep hours.
- Switch Save label to "Update check-in" when today's wellness pre-filled the form; show success as a brief toast or button-morph-to-checkmark.
- Read the weight unit from the athlete profile when available.

**OpenSpec change:** `quick-checkin-form`
