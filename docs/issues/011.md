# 011 — Live refresh while workout analysis runs

**Area:** detail · **Priority:** medium · **Effort:** M · **Status:** open

## Problem
"Analyzing…" is static; the user must leave and re-enter the screen to see results even though the backend emits `WORKOUT_ANALYSIS_READY` push.

## Proposal
While `analysis.phase === 'analyzing'`, poll `useActivitySummaryQuery` (refetchInterval ~5s) and/or invalidate on the push event via the existing push-handling pipeline. Add a subtle shimmer on the analysis block during the wait.
