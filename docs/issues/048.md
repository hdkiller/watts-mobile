# 048 — Activity detail: analysis wall, scale-less scores, chart/desc polish

**Area:** detail · **Priority:** medium · **Effort:** M · **Status:** open

## Problem
Verified live on an analyzed ride ([activity/[id].tsx](<../../app/(app)/activity/[id].tsx>)):

1. **Scores have no scale or shape.** "Overall 8 · Technical 5 · Effort 9 · Pacing 5 · Execution 7" render as bare numbers; nothing says /10, and the `flex-wrap min-w-[64px]` layout wraps unevenly (one score orphaned on the second row).
2. **The AI analysis is an unbroken prose wall.** Executive summary + every section + recommendations + strengths + focus areas all render expanded — several screens of text sitting **above** the map, charts, and zone bars. For a field companion the glanceable objective data is buried under the essay.
3. **Chart title lies when a series is missing.** "Power & heart rate" with a legend of only "Heart rate (bpm)" on a ride without power. Title should be built from the series actually present.
4. **Raw import text as body prose.** The description ("-- Intervals icu -- Wind: NW 17–24 km/h … Device: Apple Watch Ultra") renders as regular paragraph text at the bottom — useful data, wrong presentation.

## Proposal
- Scores: fixed 5-column row (or 2×3 grid), value as "8**/10**" with the denominator muted, or a small ring/bar per score using the existing zone/brand palette.
- Reorder: hero + summary grid → scores + executive summary → **map + charts + zone bars** → collapsible "Full analysis" (sections, recommendations, strengths/focus collapsed behind a disclosure, remember-open not required).
- Chart title from available series (`Heart rate` / `Power` / `Power & heart rate`).
- Detect import-style descriptions (leading `--` marker or key:value lines) and render them as a muted "Import notes" disclosure instead of body prose.
