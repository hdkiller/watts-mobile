# 044 — Notifications inbox: repetition wall and inconsistent timestamps

**Area:** notifications · **Priority:** low · **Effort:** S · **Status:** done

## Problem
Verified live: the inbox is a wall of near-identical "Workout Analysis Ready" cards (same title, same body shape, five in the first viewport). Timestamps mix formats — "2d ago" on recent items, then bare "6/5/2026" (US order, regardless of locale) further down. Scanning value is low and the date jump feels like a different app.

## Proposal
One timestamp rule: relative up to 7 days ("2d ago"), then locale-aware short date (`toLocaleDateString` with weekday/month/day — same format `formatActivityDate` already uses). Collapse consecutive same-type/same-workout notifications or at least visually de-emphasize repeats (muted title after the first). Make rows tappable through to the workout they reference (data already carries the path — `resolvePushOpen`).
