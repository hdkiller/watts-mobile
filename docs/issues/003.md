# 003 — Recommendation hero: state color, confidence visual, accepted state

**Area:** Today · **Priority:** high · **Effort:** M · **Status:** done

## Problem
"Today's call" always renders in brand green regardless of the decision; confidence shows as debug-like text ("Confidence 85%"); after accepting, the only change is a small green "Accepted" label.

## Proposal
In `app/(app)/(tabs)/today.tsx`:
- Tint the hero by action: green for train, calm blue/violet for rest, amber for modify.
- Replace the confidence sentence with a quiet visual (3 dots or thin bar); consider hiding above ~50%.
- On accept, swap the CTA block to a confirmed state (checkmark + "Accepted — view workout").

**OpenSpec change:** `recommendation-hero-states`
