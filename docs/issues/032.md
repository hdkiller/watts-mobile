# 032 — Health sleep prefill can double-count: window spans two nights and overlapping samples are summed

**Area:** Log/health · **Priority:** medium · **Effort:** M · **Status:** done

## Problem
[healthPrefill.ts](../../src/features/log/healthPrefill.ts) sums asleep time over the window **yesterday 00:00 → now**:

1. **Two nights in one window.** At 8am the window still contains yesterday's early-morning tail of the *previous* night (e.g. 00:00–07:00 yesterday) plus all of last night — the sum can exceed 12–14h for a normal sleeper.
2. **Overlapping samples are added, not merged.** HealthKit routinely has concurrent sleep samples from multiple sources (Apple Watch stages + iPhone in-bed detection, third-party apps). Each `ASLEEP_VALUES` sample's duration is added independently, so 7h of sleep recorded by two sources prefills as ~14h. Health Connect (`pageSize: 5` sessions) has the same additive logic.

The athlete is asked to review before saving, but a plausibly-wrong number (8.2 vs 7.4) will get saved as fact and feeds recovery guidance.

## Proposal
Anchor the window to "last night": e.g. yesterday 18:00 → now (or the most recent contiguous sleep block). Merge intervals before summing (sort by start, coalesce overlaps) instead of adding durations; on HealthKit prefer a single source (or stage samples only) when both stage and in-bed samples exist. Unit-test with overlapping fixtures.
