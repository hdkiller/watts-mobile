# 063 — Health Sync provenance + workout `platformSessionId` idempotency

**Status:** done in unstaged mobile + coach-wattz implementation (2026-07-21)  
**Area:** health sync / API  
**Related:** `openspec/changes/health-platform-ingest`, `openspec/changes/health-platform-ingest-v2`

## Problem

### Wellness `lastSource`

Mobile Health Sync uploads via `POST /api/wellness` with `rawJson.source` = `healthkit` | `health_connect`. The wellness repository sets `lastSource` from the auth context (`oauth:<appId>` for companion OAuth), not from the request body. Backend merge rules that key off `lastSource` (Garmin / Whoop / etc.) therefore cannot distinguish Health Sync writes from other OAuth clients without reading `rawJson`.

### Workout upload duplicates

Mobile `POST /api/workouts/upload-fit` already sends metadata:

```json
{ "source": "healthkit"|"health_connect", "platformSessionId": "<stable id>", ... }
```

Without server-side idempotency keyed on `(source, platformSessionId)`, re-uploads after a reinstall, second device, or failed client presence-match can create duplicate workouts.

## Desired

1. Allow mobile (or any OAuth client) to pass a stable platform source that lands in `lastSource` (or a dedicated field), e.g. `healthkit` / `health_connect`, while keeping `rawJson` for reprocessing. Then apply source-aware merge so Health Sync does not thrash richer wearable fields when product wants that.
2. Make `upload-fit` idempotent on `(source, platformSessionId)` — return the existing workout id when the pair already exists (and treat as success / `duplicate: true`).

## Resolution

- Official mobile OAuth wellness writes now stamp `lastSource` from the validated `healthkit` / `health_connect` provenance marker.
- FIT ingestion receives a stable `health_<platform>_<platformSessionId>` external ID. The upload endpoint resolves an existing workout before queueing and re-triggers an unlinked duplicate FIT safely.
- The client treats new uploads as queued until `GET /api/workouts` confirms the stable external ID and then records `remoteWorkoutId`.
