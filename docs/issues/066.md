# 066 — Today / activation → Health Sync & Connected Apps: Back leaves the setup path

**Area:** navigation / activation · **Priority:** high · **Effort:** M · **Status:** open

## Problem
Several setup entry points `router.push` into the **More → Settings** stack:

| From | Target |
|------|--------|
| Today `FinishSetupCard` | `APP_HREFS.settingsHealth` / `settingsConnectedApps` |
| Today “Connect a device” cue | `settingsConnectedApps` |
| Activation `/(activation)/connect` | same settings hrefs |

Back walks Settings (anchored at settings index) → More, not Today or the activation wizard. Soft-activated and day-one athletes hit this on the critical connect path.

## Proposal
Present Health Sync / Connected Apps as **root-stack** (or modal) screens when opened outside More — same multi-entry pattern as `athlete` / `daily-checkin` / `recovery-event`. Intra-More Settings navigation can keep the nested stack. Update `APP_HREFS` and any deep links in the same change.
