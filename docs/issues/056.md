# 056 — First-run experience unaudited: day-one athlete sees mostly empty surfaces

**Area:** onboarding · **Priority:** medium · **Effort:** M · **Status:** open

## Problem
All review passes so far ran against a mature account (weeks of activities, profile, AI history). A brand-new athlete — fresh sign-up, no synced device, no plan — hits a very different app on day one, and that path decides retention:

- Today: no recommendation, no planned workout, empty wellness/load/week sections stacked as a column of "No X yet" cards; the strongest CTA ("Analyze Readiness") may produce a weak recommendation with zero biometrics behind it.
- Log/recovery/measurements: all empty states simultaneously; Coach chat has no context to seed.
- Nothing explains the dependency: connect a device / set up a plan **on the web** → then the companion comes alive. The empty states are individually honest (per DESIGN.md) but collectively say "nothing works yet" without pointing at the one action that fixes it.

## Proposal
Audit the app against a fresh account (e2e seed with an empty fixture user). Then design one first-run state for Today that replaces the pile of empty sections: a short "Get set up" card sequence (connect integrations on web → plan → first check-in) that disappears once data exists. Verify the store-review experience too — a reviewer's account is a day-one account.
