# 056 — First-run experience unaudited: day-one athlete sees mostly empty surfaces

**Area:** onboarding · **Priority:** high · **Effort:** L · **Status:** open

**Superseded in spirit by** [product-baseline.md](../product-baseline.md) § Activation onboarding (2026-07-21 reposition). Keep this issue until the activation wizard + Finish-setup Today ship.

## Problem
All review passes so far ran against a mature account (weeks of activities, profile, AI history). A brand-new athlete — fresh sign-up, no synced device, no plan — hits a very different app on day one, and that path decides retention:

- Today: no recommendation, no planned workout, empty wellness/load/week sections stacked as a column of "No X yet" cards; the strongest CTA ("Analyze Readiness") may produce a weak recommendation with zero biometrics behind it.
- Log/recovery/measurements: all empty states simultaneously; Coach chat has no context to seed.
- Nothing explains the dependency or offers in-app activation. The empty states are individually honest (per DESIGN.md) but collectively say "nothing works yet."

## Proposal
Ship the **activation companion** path (baseline): server-driven wizard (consent → goal lite → plan lite → first insight → connect data last), then replace stacked empties with a single Finish-setup / resume surface until fully activated. Audit with an empty fixture user and store-review accounts.
