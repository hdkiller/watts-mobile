# 041 — Implausible health values shown prominently ("Sleep 0.2 hrs ↓ 69%")

**Area:** Today/wellness · **Priority:** medium · **Effort:** S · **Status:** open

## Problem
Verified live: the Recent Wellness glance on Today headlines "**Sleep 0.2 hrs**" with a red "↓ 69%" badge. A 12-minute sleep reading is almost certainly a sync artifact (partial HealthKit sample, watch not worn), yet it gets hero treatment plus an alarming trend badge. Garbage-in rendered confidently erodes trust in every other number the coach shows, and the derived trend (% vs an equally broken baseline) compounds it.

## Proposal
Plausibility-gate glance metrics before display (in `useRecentWellness` / the glance mapper): sleep < 1h or > 16h, RHR < 25 or > 120, weight jumps > 10%/day → show "No sleep data" / "—" with muted styling instead of value + trend. Suppress the trend badge whenever either endpoint is implausible. Same gate should apply anywhere these metrics surface (wellness overview sheet, seed context sent to the coach).
