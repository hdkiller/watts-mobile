# 068 — Notification inbox / push opens nested tab stacks without a return path

**Area:** navigation / notifications · **Priority:** medium · **Effort:** M · **Status:** open

## Problem
1. **Inbox:** `openNotificationLink` does `router.push(resolved.href)` into Today (activity/planned), Coach, etc. Back lands on that tab’s stack root, not the inbox. Inbox already special-cases its own header Back to More (`goBackToMore`); outbound opens do not.
2. **Push / universal links (warm start):** `PushNotificationsBootstrap` uses `withAnchor: true`, so nested More/Settings targets get a tab root under them — but the previously focused tab (e.g. Today) is not restored on Back. Today’s stack has no `unstable_settings.anchor`.

Cold-start deep links landing on a tab root are fine; mid-session jumps that strand Back are the defect.

## Proposal
Multi-entry detail destinations that inbox and push can open should follow the root-stack pattern (or push with an explicit return href / custom Back to inbox). Revisit Today `unstable_settings.anchor = 'index'` for nested Today deep links. Keep path map (`pathMap` / `resolveDeepLink`) stable.
