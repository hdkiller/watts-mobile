# 068 — Notification inbox / push opens nested tab stacks without a return path

**Area:** navigation / notifications · **Priority:** medium · **Effort:** M · **Status:** done

## Problem
Inbox and push opened activity/planned (etc.) under the Today tab stack, so Back left the inbox. Warm-start nested More/Settings jumps had the same class of bug.

## Proposal
Multi-entry destinations on the root stack; migrate legacy Today/More nested hrefs from push payloads.

## Resolution (2026-07-23)
Covered by the root-stack hoist in **065–067**: inbox `router.push(resolved.href)` and push `migrateLegacyAppHref` now land on `/(app)/activity|planned|…` / health / sports. Back returns to the inbox (or prior tab surface) under `(tabs)`. Notifications remain under More (same-tab opens).
