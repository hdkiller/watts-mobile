# 046 — Small-polish sweep: double Settings title, placeholder-as-value fields, "Water 0.0 L"

**Area:** cross-cutting · **Priority:** low · **Effort:** S · **Status:** done

## Problem
Grab-bag of live-observed paper cuts, individually tiny, together "unfinished":

1. **Settings shows its title twice** — native header "Settings" plus an in-content `text-2xl` "Settings" heading directly below ([settings/index.tsx](<../../app/(app)/settings/index.tsx>)). Stack screens elsewhere use one or the other.
2. **Nutrition quick-log placeholders look like values** — Calories/Protein/Carbs/Fat show "320 / 25 / 30 / 10" as placeholders; at a glance the form looks pre-filled, and clearing-vs-typing intent is ambiguous. Use unit hints ("kcal", "g") or empty fields with labels carrying the unit.
3. **"Water 0.0 L"** renders under "No fuel logged yet today" in the Today nutrition glance — a zero reading is noise when nothing is logged; show it only once something was.
4. **Double titles repeat across settings screens** — Health Sync and Units & locale also render native header + identical in-page h1 (same pattern as Settings hub).
5. **Upcoming rows repeat the full date under day headers** — a row under "TOMORROW" still says "Tue, Jul 21 · Run · 30 min"; the date column is redundant inside a day-grouped list.

## Proposal
Fix all five in one pass: drop the in-content headings on settings screens (keep subtitle lines), swap numeric placeholders for unit hints, hide the hydration line at zero, and drop the date from row meta on the grouped Upcoming list (keep type · duration · TSS).
